# データ作成メモ

## プロジェクトの背景

学部ゼミ（2026年度）の講義資料として、田町駅周辺の飲食店（ラーメン屋・牛丼屋）の分布を
2016年と2026年で比較する地図を作成する。元々は食べログの地図検索画面のスクリーンショット
（Google Maps埋め込み）を使用していたが、これを独自の地図マッピングに置き換える。

## データ取得の技術的課題と解決

### 課題1: 2016年データの取得

2016年のデータはPDFスクリーンショット（食べログの検索結果地図）しか残っていない。

**解決方法: 画像処理 + 食べログ閉店ページ**

1. **画像からのピン検出** (`detect_pins.py`)
   - PDFを300dpiで画像化
   - 地図部分を切り出し
   - OpenCVのHSV色空間でオレンジピン（食べログのマーカー）を検出
   - パラメータ: H=8-22, S>160, V>180（道路の黄色と区別）
   - 形状フィルタ: circularity > 0.15（細長い道路を除外）

2. **ジオリファレンシング** (`georef.py`)
   - 地図上の既知ランドマーク4点の (ピクセル, 緯度経度) 対応
     - 三田駅 (緑アイコン): (797, 523) → (35.64880, 139.74690)
     - 田町駅西口 (テキスト): (575, 692) → (35.64600, 139.74620)
     - JR田町 (青アイコン): (668, 812) → (35.64540, 139.74760)
     - 三田通り交差前 (テキスト): (400, 210) → (35.65090, 139.74370)
   - 最小二乗法でアフィン変換パラメータを推定
   - 精度: RMS ≈ 15m
   - 検証: ラーメン二郎三田本店 → 最近接ピンまで34m、むらさき山 → 18m

3. **食べログからの補完** (`geocode_2016_from_tabelog.py`)
   - 食べログでは閉店した店舗のページが【閉店】タグ付きで保存されている
   - 各ページのHTML内に `lat:`, `lng:` が記載 → 正規表現で抽出
   - 閉店ページURL例: `tabelog.com/tokyo/A1314/A131402/13004559/`

4. **最終データ構築** (`build_2016_dataset.py`)
   - 画像抽出座標 + 食べログ座標 + 既知住所を統合
   - 田町駅1200m以内の座標チェックで全件検証

### 課題2: 2026年データの取得

**解決方法: 食べログスクレイピング** (`tabelog_scraper.py`)

- URL構造: `tabelog.com/tokyo/A1314/A131402/rstLst/{ジャンルコード}/`
  - ラーメン: `MC0101`
  - 牛丼: `RC011101`
- 一覧ページ → 店舗URL取得 → 詳細ページから座標抽出
- 詳細ページのHTML内に `lat: 35.xxxxx, lng: 139.xxxxx` が存在
- アクセス間隔: 3秒
- robots.txt上、一覧・詳細ページはDisallowされていない

### ジオコーディングの注意

- Nominatim (OpenStreetMap) は日本の番地レベル住所（「港区芝5-25-9」形式）に弱い
- 食べログ詳細ページからの直接座標取得が最も正確
- 食べログURLの店舗IDは推測できないため、一覧ページからの取得が必要

## データの信頼性

| データ | 精度 | 注意点 |
|--------|------|--------|
| 2016年 画像抽出座標 | ≈15m | アフィン変換による推定値。参照点精度に依存 |
| 2016年 食べログ座標 | ≈5m | 食べログに登録された正確な座標 |
| 2016年 既知座標 | ≈30m | 有名店の住所から推定。完全には正確でない可能性 |
| 2026年 食べログ座標 | ≈5m | 食べログに登録された正確な座標 |
| 2016年 店舗リスト | 27/31店 | PDFに全31件と表示。27店を復元（87%） |
